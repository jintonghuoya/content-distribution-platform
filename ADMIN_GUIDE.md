# CDP 管理后台使用手册

## 快速启动

### 1. 环境准备

确保已安装：
- Docker Desktop
- Git

### 2. 配置环境变量

编辑项目根目录的 `.env` 文件：

```env
# 必填：Claude API Key（用于内容生成）
LLM_API_KEY=sk-ant-xxxxx

# 以下默认即可，一般不需要改
DATABASE_URL=postgresql+asyncpg://cdp:cdp@postgres:5432/cdp
REDIS_URL=redis://redis:6379/0
CELERY_BROKER_URL=redis://redis:6379/0
CELERY_RESULT_BACKEND=redis://redis:6379/0
```

### 3. 启动服务

```bash
cd /gt/cdp/mayor/rig
docker compose down && docker compose build && docker compose up -d
```

### 4. 初始化数据库

```bash
docker compose exec api alembic upgrade head
```

### 5. 打开管理页面

浏览器访问：**http://localhost:8088**

## 页面功能说明

### 控制面板 (`/`)

- 顶部 5 张统计卡片：显示各阶段话题数量（待过滤/已过滤/生成中/已发布/已拒绝）
- 中间流水线操作区：5 个按钮分别触发「采集→过滤→生成→分发→收益」全流程
- 底部最近话题：最新 10 条话题列表

### 数据源 (`/sources`)

管理采集来源（微博/百度/知乎等）。

| 操作 | 说明 |
|------|------|
| 开关切换 | 启用/禁用某个采集源 |
| 立即采集 | 手动触发一次全量采集 |

### 话题管理 (`/topics`)

浏览和操作热点话题。

| 功能 | 说明 |
|------|------|
| 筛选 | 按来源（微博/百度等）和状态（待过滤/已过滤等）筛选 |
| 分页 | 支持 20/50/100 条每页 |
| 详情 | 点击「详情」查看话题完整信息和原始数据 |
| 过滤 | 对 pending 状态的话题执行过滤规则 |
| 生成 | 对 filtered 状态的话题触发 AI 内容生成 |

### 过滤规则 (`/filters`)

管理话题过滤规则。

**内置规则类型：**
- **关键词黑名单**：包含指定关键词的话题会被过滤掉
- **热度阈值**：只保留热度在指定范围的话题
- **来源权重**：不同来源赋予不同权重系数

| 操作 | 说明 |
|------|------|
| 新增规则 | 点击「新增规则」，选择类型，填写配置 |
| 编辑规则 | 点击「编辑」修改已有规则 |
| 启用/禁用 | 用开关控制规则是否生效 |
| 全量过滤 | 对所有 pending 话题执行一遍过滤 |

### 内容管理 (`/content`)

查看和管理 AI 生成的文章。

| 操作 | 说明 |
|------|------|
| 批量生成 | 对所有 filtered 话题触发内容生成 |
| 预览 | 点击「预览」查看生成的文章内容 |
| 分发 | 选择平台后分发单条内容（已分发的内容自动跳过） |
| 批量分发 | 勾选多条内容，选择平台批量分发 |

**防重复**：同一条内容在某个平台已成功分发后，再次点击分发会自动跳过。

### 分发记录 (`/distribution`)

查看内容分发到各平台的历史记录。

| 操作 | 说明 |
|------|------|
| 平台筛选 | 按平台（微博/微信等）筛选分发记录 |
| 内容包 | packaged 模式的记录可查看格式化内容，手动复制发布 |

### 收益统计 (`/revenue`)

查看各平台内容的收益数据。

- 顶部 4 张汇总卡片：总收益/总浏览/总点赞/总互动
- 中间收益趋势图：柱状图+折线图
- 底部收益明细表：逐条记录
- 「采集收益」按钮触发收益数据抓取

### 平台配置 (`/settings`)

配置各平台的发布凭据。

**支持 6 个平台：** 微博、微信公众号、今日头条、小红书、B站、抖音

每个平台可配置：
- **Cookie**：从浏览器复制的 Cookie 字符串
- **API Key**：平台提供的 API 密钥
- **App ID**：应用 ID
- **App Secret**：应用密钥

点击展开对应平台的面板，填写后点击「保存配置」。

### Playwright Browser Service（宿主机）

微博和小红书通过宿主机上的 Playwright 服务自动发布，需单独配置：

```bash
cd browser-service
pip install -r requirements.txt
playwright install chromium
python server.py          # 启动服务（端口 8001）
```

首次使用需保存登录态：
```bash
python login.py weibo        # 浏览器中手动登录，保存 cookie
python login.py xiaohongshu  # 同上
```

## 推荐启动流程

第一次使用时，按以下顺序操作：

1. **数据源** → 确认采集源已启用，点「立即采集」
2. **话题管理** → 确认采集到了话题数据
3. **过滤规则** → 检查默认规则是否合适，可新增自定义规则
4. **话题管理** → 选一条话题点「过滤」，或在过滤规则页点「全量过滤」
5. **话题管理** → 对已过滤的话题点「生成」
6. **内容管理** → 预览生成的内容
7. **平台配置** → 填入目标平台的凭据（微博/小红书需先在宿主机启动 Browser Service 并保存登录态）
8. **内容管理** → 点「分发」选择目标平台发布
