## 方案二：技术分享视角

### 标题
**用Claude API搭了一个全自动内容工厂，架构分享**

### 封面建议
- 主图：系统架构图（流程图风格），节点用不同颜色区分模块
- 风格：技术文档风，白底/浅灰底 + 彩色模块
- 加文字："5个模块 | 全自动 | 开源"

### 正文
分享一个刚做完的开源项目：CDP（Content Distribution Platform）

一句话概括：**热点话题感知 → 内容过滤 → AI生成 → 多平台分发 → 收益追踪**

5个模块，全自动串起来：

```
Collector → Filter → Generator → Distributor → Revenue
热点采集    内容过滤   AI内容生成   多平台分发     收益追踪
```

几个技术亮点：

1️⃣ **采集器注册机制**
BaseCollector + Registry 模式，新增平台只需写一个类，自动发现注册

2️⃣ **多层过滤管道**
关键词黑名单 → 热度阈值 → AI审核，三层过滤保证内容质量

3️⃣ **适配器模式分发**
每个平台一个Distributor，统一 publish() 接口，微信/微博已实现，小红书用Playwright自动化

4️⃣ **Claude API 驱动生成**
用 Claude Sonnet 生成文章和社交文案，temperature 区分（文章0.8/文案0.9）

技术栈：FastAPI + Celery + Redis + PostgreSQL + SQLAlchemy + Claude API
Docker Compose 一键启动，开箱即用

GitHub 开源，欢迎 star 和 PR 🌟
有问题评论区交流～
